# 小程序模块范围

## 产品定位

小程序不是 Web 后台的缩小版，定位为移动端工作台，服务现场值守、巡检和轻量调度。

适合放在小程序的能力：

- 高频查看：当天灌溉、地块状态、设备健康度、告警
- 轻量操作：确认、启停、查看执行结果
- 现场协同：巡检、扫码、快速核对设备与地块状态

不适合直接搬到小程序的能力：

- 复杂地图编辑和地块绘制
- 大批量计划编排
- 完整策略设计器
- 系统级配置、权限和治理后台

## 模块划分

### 1. 总览

目标：一屏判断今天是否需要处理异常、补灌或调整计划。

包含：

- 总指标：地块数、在线设备、待执行计划、供水负荷
- 地图预览：地块位置分布、状态颜色、只读查看
- 调度建议：基于天气、墒情、计划窗口给出当天建议
- 高风险地块：按风险排序
- 即将执行：最近计划窗口

不做：

- 地图编辑
- 复杂报表
- 长周期经营分析

### 2. 地块

目标：围绕单个地块看状态、分区和近期执行情况。

包含：

- 地块列表：状态、作物、面积、墒情、分区数
- 地块详情：作物、生育期、墒情、建议时长
- 分区状态：站号、土壤湿度、待机/待执行/执行中/告警

二期可补：

- 地块只读地图
- 最近执行记录
- 临时灌溉入口

不做：

- 地块绘制
- 分区边界编辑
- 复杂设备绑定调整

### 3. 计划策略

目标：把执行计划和自动决策放在一个入口，减少移动端导航深度。

标签页一：计划

- 计划列表
- 计划窗口、分区数、总时长
- 执行模式：自动 / 确认 / 手动
- 近期执行状态

标签页二：策略

- 策略列表
- 策略类型：阈值 / ET
- 作用范围：整块地 / 分区
- 模式：自动 / 确认 / 建议
- 雨锁、最小间隔等关键信息

移动端保留操作：

- 启停
- 确认执行
- 忽略本次建议
- 查看最近状态

保留在 Web：

- 复杂计划创建与编排
- 完整策略设计和大量参数编辑

### 4. 设备

目标：服务现场巡检和链路状态核对。

包含：

- 设备列表：控制器、传感器
- 状态筛选：在线 / 离线 / 告警
- 关键指标：信号、电量、最后在线时间
- 关联信息：所属地块、通道数

二期可补：

- 设备详情
- 控制回执
- 扫码识别
- 调试入口

### 5. 我的

目标：承接用户、项目和运行配置，不混入核心业务流程。

包含：

- 用户信息
- 项目/农场切换
- 通知设置
- 版本信息
- 运行环境与接口诊断

当前一期先保留：

- 数据源状态
- Node 服务接入信息

## 一期实现顺序

1. 五个模块导航结构稳定
2. 总览数据模型稳定
3. 地块、计划策略、设备列表页接真实读接口
4. 登录态、项目切换
5. 启停、确认等轻操作

## 接口原则

- 小程序统一走 Node 服务，不直连 Supabase
- 共用领域类型和纯函数放在 `packages/irrigation-domain`
- 页面只消费 view model，不直接拼装复杂业务规则
