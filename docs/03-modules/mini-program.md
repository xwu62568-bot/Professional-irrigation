# 小程序模块

## 定位

小程序是移动端现场工作台，服务巡检、轻量调度、异常确认和 AI 问答，不是 Web 后台的缩小版。

## 当前能力

- 登录和 mini token 存储。
- 总览、地块、计划策略、设备、我的、AI 助手页面。
- 地块/设备/计划/策略读取真实 mini API。
- 计划创建、编辑、启停。
- 计划列表与计划详情页在可见态自动刷新执行状态（串行静默刷新，避免并发请求堆积）。
- 列表页按标签页自适应刷新周期（计划约 8 秒、策略约 15 秒）。
- 启动/暂停/停止计划后会立即触发一次静默刷新，缩短状态收敛时间。
- 演示设备详情控制。
- AI 助手流式对话。

## 当前不标准点

- 策略创建调用服务端后仍未真实写库。
- 策略确认/忽略存在演示提示。
- 真实设备控制尚未生产化。
- 端侧部分 view model 转换用于兼容演示阶段。

## 后续标准做法

- 小程序继续统一走 Node API，不直连 Supabase。
- 小程序只做移动端适合的轻操作。
- 复杂地图绘制、批量编排、系统治理留在 Web。
- 每个 mini API 必须在 `packages/irrigation-api` 定义契约。
