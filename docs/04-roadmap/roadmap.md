# 路线图

## 阶段 0：文档和目标对齐

目标：建立标准项目文档体系，明确演示态、生产态、缺口和迭代追踪。

验收：

- 文档入口完整。
- 缺口分析可追踪。
- 生产化目标架构明确。

## 阶段 1：业务闭环补齐

目标：补齐策略 CRUD、策略动作、计划执行测试、设备绑定可靠性。

重点：

- 策略创建/启停/确认/忽略真实写库。
- 小程序和 Web 策略表现一致。
- 计划执行状态机有测试。

## 阶段 2：统一 API 和数据一致性

目标：减少 Web 直连 Supabase 写业务的双路径。

重点：

- 设计服务端业务 API。
- Web 写操作逐步迁移。
- 共享 DTO 和契约测试。
- 已完成：灌溉计划 CRUD 与启动入口已统一到 `execution-service /mini/plans*`，Web 改计划时间会同步更新 `plan_schedule_jobs`。

## 阶段 3：设备链路生产化

目标：从固定演示设备升级为正式设备接入能力。

重点：

- 设备状态和事件落库。
- ACK、超时、重试、失败原因。
- demo 链路隔离。

## 阶段 4：权限、监控和发布治理

目标：达到可生产运营的最低标准。

重点：

- 项目/组织/角色权限。
- 审计日志、告警、监控。
- 发布、回滚、备份和恢复流程。
